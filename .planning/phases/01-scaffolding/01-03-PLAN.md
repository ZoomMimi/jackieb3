<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local repo → Netlify build runtime | Netlify runs `npm install` + `npm run build` on its infrastructure |
| Netlify CDN → public visitors | Static assets served over HTTPS to anonymous users |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Tampering | Netlify build (runs npm install on push) | mitigate | package-lock.json pins exact dependency versions; Node pinned to 22 via .nvmrc + netlify.toml so the build environment is deterministic |
| T-01-07 | Information Disclosure | Repo contents on Netlify build | accept | No secrets in repo; no env vars required for a static build (user_setup env_vars empty). .planning/data/ files are gitignored / not in dist |
| T-01-08 | Spoofing | Live site over HTTP | mitigate | Netlify serves the production URL over HTTPS by default with automatic TLS — verified in the human checkpoint (step 3) |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | No new package-manager installs in this plan; deps were audited/installed in 01-01. N/A |
</threat_model>
