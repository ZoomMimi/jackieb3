# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind 4 utility classes directly (D-01) |
| Preset | not applicable |
| Component library | none — no React/shadcn; Astro components only |
| Icon library | none — Phase 1 is stub pages; no icons required |
| Font | Lora (serif, body) + Inter (sans-serif, headings/UI) via Google Fonts (D-04, D-05) |

**Loading strategy:** Use Astro 6 built-in Fonts API (`fontProviders.google()`) in `astro.config.mjs`. If still behind an `experimental` flag at build time, fall back to standard `<link rel="preconnect">` + `<link href="...fonts.googleapis.com/css2?...">` tags in `BaseHead.astro`. Either path satisfies D-05.

**Tailwind configuration:** All color tokens declared in `src/styles/global.css` via `@theme {}` block — no `tailwind.config.mjs` file (Tailwind 4 is CSS-native; config files are a Tailwind 3 artifact).

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (nav link internal gaps) |
| sm | 8px | Compact element spacing (tag chips, caption padding) |
| md | 16px | Default element spacing (paragraph gap, card padding) |
| lg | 24px | Section padding (header/footer padding, card container gap) |
| xl | 32px | Layout gaps (main content top margin, section breaks) |
| 2xl | 48px | Major section breaks (hero bottom, between homepage sections) |
| 3xl | 64px | Page-level spacing (top-of-page padding on desktop) |

Exceptions: Nav touch targets on mobile set to minimum 44px height (accessibility — iOS tap target guideline). Implemented via `min-h-[44px]` Tailwind utility.

## Typography

### Font Assignments

```css
/* In src/styles/global.css */
body        { font-family: var(--font-lora), Georgia, serif; }
h1, h2, h3, h4, nav, button, label { font-family: var(--font-inter), system-ui, sans-serif; }
```

### Type Scale

Exactly 4 declared sizes. Exactly 2 declared weights.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Label | Inter (sans) | 14px | 400 (regular) | 1.5 | Meta info: date, location, voyage tag |
| Body | Lora (serif) | 18px | 400 (regular) | 1.7 | Post body text, paragraph prose; also UI/Nav links at 18px |
| Heading 2 | Inter (sans) | 20px | 700 (bold) | 1.3 | Post section headings, card titles, page sub-headings, nav links, CTAs |
| Heading 1 | Inter (sans) | 28px | 700 (bold) | 1.2 | Page titles, post titles, hero heading |

**Display exception (not a scale member):** The homepage hero title "Going Loopy" uses `text-[36px]` as a Tailwind arbitrary value — a one-off exception applied only to that single element. It is not part of the declared type scale.

**Rule:** Nav links, button labels, CTA link text, and card titles all use 18px (body/UI size) or 20px (Heading 2 size) at weight 700. There is no semibold (600) weight in this project.

**Italic:** Lora italic (weight 400) loaded for block quotes and pull quotes in post body. Not used in Phase 1 stubs but the font load should include it so Phase 2/3 doesn't require a config change.

## Color

### Palette

| Role | Token | Hex | Tailwind Usage |
|------|-------|-----|----------------|
| Dominant (60%) | `--color-offwhite` | `#faf9f6` | Page background, main content area |
| Secondary (30%) | `--color-navy` | `#1a2e4a` | Navigation bar, footer, card borders, horizontal rules |
| Accent (10%) | `--color-brass` | `#c9a84c` | See accent list below |
| Text primary | `--color-ink` | `#1c1c1e` | All body copy and headings on off-white background |
| Text secondary | `--color-muted` | `#6b7280` | Meta labels (date, location, voyage tag) |
| Destructive | `--color-danger` | `#b91c1c` | Destructive actions only (none in Phase 1) |

**Source:** D-03 provides `#1a2e4a`, `#faf9f6`, `#c9a84c` as the anchoring values. `--color-ink` and `--color-muted` are defaults added to ensure readable body copy on the off-white background without using pure black.

### Accent Reserved For

The brass/gold accent (`#c9a84c`) is reserved exclusively for:
1. Inline hyperlinks in body prose (`<a>` tags within post content)
2. The site name / logo text in the nav header
3. The active/current nav link underline indicator
4. "Read more" and CTA link text on the homepage recent-posts list
5. Horizontal rule decorators (`<hr>`) used as section dividers

Accent is NOT used for: backgrounds, cards, buttons, borders, icons, or any decorative element beyond the five items above.

### Navy Usage (Secondary 30%)

Navy (`#1a2e4a`) is used as the background color for:
- `<header>` / nav bar (full-width dark bar, white text on navy)
- `<footer>` (full-width dark bar, white text on navy)
- Section divider lines (1px `border-color: #1a2e4a`)

Text on navy surfaces uses white (`#ffffff`), not off-white.

### CSS Custom Properties

```css
/* src/styles/global.css */
@import "tailwindcss";

@theme {
  --color-navy:     #1a2e4a;
  --color-offwhite: #faf9f6;
  --color-brass:    #c9a84c;
  --color-ink:      #1c1c1e;
  --color-muted:    #6b7280;
  --color-danger:   #b91c1c;
  --font-lora:      'Lora', Georgia, serif;
  --font-inter:     'Inter', system-ui, sans-serif;
}

body {
  background-color: var(--color-offwhite);
  color: var(--color-ink);
  font-family: var(--font-lora);
  font-size: 18px;
  line-height: 1.7;
}
```

## Layout Contracts

### Navigation (D-07)

**Desktop (≥768px):**
- Full-width `<header>` with navy background
- Site name/logo text left — "JACKIE B III" in Inter 700, brass color, 18px
- Nav links right — "Home", "Great Loop", "About" in Inter 700, white, 18px
- Active link: brass underline (2px solid), white text remains
- Header height: 64px, vertical padding 12px top + bottom

**Mobile (<768px):**
- Same navy header, 56px height
- Site name left, hamburger icon right (3 horizontal lines, white, 24px)
- Hamburger toggles a full-width dropdown nav panel (navy background, links stacked, 44px min-height each)
- No JS animation required in Phase 1 — a simple `<details>`/`<summary>` toggle is acceptable as the mobile nav implementation

**Nav links order:** Home → Great Loop → About

### Homepage Layout (D-06)

Three stacked sections, no sidebar:

1. **Hero block** — full-width, navy background, centered text
   - Display heading: "Going Loopy" — Inter 700, `text-[36px]` (one-off exception, not a scale member), off-white
   - Subhead: "Jackie B III on the Great Loop, 2022–2024" — Inter 400, 18px, muted
   - CTA link: "Read the voyage" → `/voyages/great-loop/` — brass, Inter 700, 18px
   - Padding: 64px top/bottom desktop, 48px mobile

2. **Recent Posts list** — centered container, max-width 720px
   - Section heading: "Recent Posts" — Inter 700, 20px, ink
   - Up to 5 post cards (stub: dummy post only in Phase 1)
   - Card layout: date label (muted 14px) + post title (link, brass, Inter 700, 20px) + excerpt (Lora 18px, ink) + location meta (muted 14px)
   - Cards separated by a 1px navy rule
   - "View all posts" link below the list → `/voyages/great-loop/` — brass, Inter 700, 18px

3. **Footer** — full-width, navy background
   - Copyright: "© 2022–2024 Jackie B III Going Loopy" — Inter 400, 14px, white/70% opacity
   - No social links in Phase 1

## Interaction Contracts

Phase 1 has minimal interactions — this is a static stub site:

| Interaction | Behavior | Implementation |
|-------------|----------|----------------|
| Nav link hover | Brass underline appears, text stays white | `hover:underline decoration-[#c9a84c]` Tailwind utility |
| Nav link active | Brass underline always visible | Add `aria-current="page"` on current link; style via CSS `[aria-current="page"]` |
| Mobile hamburger toggle | Dropdown nav panel shows/hides | `<details>/<summary>` element — no JS required |
| Post title link hover | Brass text darkens (10% darker) | `hover:text-[#a88a3d]` Tailwind utility |
| Focus ring | Visible on all interactive elements | Tailwind `focus-visible:ring-2 focus-visible:ring-[#c9a84c]` |

**No JavaScript required in Phase 1** beyond what Astro ships. The mobile nav using `<details>` avoids any JS dependency for the shell.

## Accessibility

- All `<img>` tags must include `alt` attribute (stub images use descriptive alt text, not empty)
- Nav must include `<nav aria-label="Main">` and `aria-current="page"` on the active link
- Hamburger toggle must include `aria-label="Toggle navigation"` and `aria-expanded` state
- Color contrast: off-white `#faf9f6` on navy `#1a2e4a` = 11.8:1 (AAA). Ink `#1c1c1e` on off-white = 18.9:1 (AAA). Brass `#c9a84c` on navy = 3.1:1 (AA Large only — use for large text/decorative links, not small body text)
- Minimum touch target: 44px height on all interactive elements (enforced via `min-h-[44px]`)

**Brass contrast note:** Brass `#c9a84c` on off-white `#faf9f6` = 2.5:1 — does NOT meet WCAG AA for body text. Brass links in prose must be at minimum 18px Lora (large text threshold) to pass AA Large. Do not use brass for label-size (14px) text. Underline decoration also helps distinguish links for low-vision users.
