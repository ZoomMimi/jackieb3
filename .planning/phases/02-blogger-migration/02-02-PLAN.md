Project-wide photo rule (STATE.md + CONTEXT.md D-05):
  Plain <img> tags with loading="lazy" decoding="async" — never <Image> from 'astro:assets'.

Decision D-08: VideoEmbed.astro accepts a YouTube URL prop (url: string), extracts the video ID,
  and constructs the embed URL as https://www.youtube.com/embed/{videoId}.
  No Blogger video token handling in this component.

Gallery.astro target spec (RESEARCH.md Pattern 5):
  Props: images: string[], caption?: string
  Grid: CSS grid-template-columns: repeat(auto-fill, minmax(280px, 1fr)), gap: 0.75rem
  Each img: width: 100%, aspect-ratio: 4/3, object-fit: cover, border-radius: 4px
  Alt text: if caption, use caption + " (N+1)"; otherwise "Photo N+1"

VideoEmbed.astro target spec (RESEARCH.md Pattern 6):
  Props: url: string, title?: string (default 'Video')
  videoId extraction regex matches both youtube.com/watch?v= and youtu.be/ URL forms
  embedUrl = videoId ? https://www.youtube.com/embed/{videoId} : url (fallback)
  iframe: src=embedUrl, title=title, frameborder="0", allowfullscreen, loading="lazy"
  allow attribute: "accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
  Wrapper div: position relative, padding-bottom 56.25%, height 0, overflow hidden, margin 1.5rem 0
  iframe positioned absolute: top 0, left 0, width 100%, height 100%, border-radius 4px, border none
