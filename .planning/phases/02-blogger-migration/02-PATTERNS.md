**Core `fetchAllPosts()` pattern** (`scripts/blog-viewer.mjs` lines 39–89):
```javascript
async function fetchAllPosts() {
  const posts = [];
  let startIndex = 1;

  while (true) {
    const url = `${FEED_BASE}&start-index=${startIndex}&orderby=published`;
    process.stdout.write(`  Fetching posts ${startIndex}–${startIndex + 24}…\r`);
    const res  = await fetch(url);
    if (!res.ok) throw new Error(`Feed returned ${res.status}`);
    const feed = await res.json();
    const entries = feed.feed?.entry ?? [];
    if (entries.length === 0) break;

    for (const entry of entries) {
      const title     = entry.title?.$t ?? '(untitled)';
      const published = (entry.published?.$t ?? '').slice(0, 10);  // "YYYY-MM-DD"
      const postUrl   = entry.link?.find(l => l.rel === 'alternate')?.href ?? '';
      const html      = entry.content?.$t ?? '';
      // import script also needs:
      // const labels = (entry.category ?? [])
      //   .filter(c => c.scheme === 'http://www.blogger.com/atom/ns#')
      //   .map(c => c.term);
      // const posterId = entry.id?.$t ?? '';
      posts.push({ date: published, title, postUrl, html /*, labels, posterId */ });
    }

    if (entries.length < 25) break;
    startIndex += 25;
  }

  posts.sort((a, b) => a.date.localeCompare(b.date));
  return posts;
}
```

### `public/_redirects` (config, request-response)

**Analog:** None in codebase. `netlify.toml` (`netlify.toml` lines 1–7) shows the existing Netlify config style but uses `netlify.toml` format, not `_redirects` format.

**File format** (standard Netlify `_redirects` syntax, from RESEARCH.md):
```
# Blogger → Astro 301 redirects (auto-generated by scripts/import-blogger.mjs)
/2022/04/getting-ready-to-go.html /blog/2022-04-16-getting-ready-to-go/ 301
/2022/04/happy-easter.html /blog/2022-04-17-happy-easter/ 301
```
Rule: `[from-path] [to-path] [status]` — one rule per line, space-separated.
The `from` path is the Blogger URL path only (no domain). Netlify matches paths, not hosts.
Decision: use `public/_redirects` not `netlify.toml` (keeps `netlify.toml` at 7 lines; see RESEARCH.md Netlify Redirect Format Decision).
Astro copies `public/` → `dist/` at build time; Netlify reads `dist/_redirects`.

### No Astro Image Component
**Source:** `STATE.md` (decision record) + `CONTEXT.md` D-05
**Apply to:** `src/components/Gallery.astro`, all 72 MDX files
```html
<!-- Use plain img, NOT <Image> from 'astro:assets' -->
<img src={url} alt="..." loading="lazy" decoding="async" />
```
Photos are served from `blogger.googleusercontent.com`. Astro's Image optimizer cannot process remote CDN URLs that lack an `inferSize` endpoint. Project decision: plain `<img>` with lazy loading.

## Key Routing Insight

**Source:** `src/pages/blog/[...id].astro` lines 1–19

```astro
export async function getStaticPaths() {
  const posts = await getCollection('blog');
  return posts.map((post) => ({
    params: { id: post.id },
    props: { post },
  }));
}
```

The `post.id` is the MDX filename without extension (e.g., `2022-04-16-getting-ready-to-go`). The `[...id]` catch-all generates URL `/blog/2022-04-16-getting-ready-to-go/`. This is why the redirect target is `/blog/${dateStr}-${slug}/` — it matches exactly what Astro generates.

The content collection loader (`src/content.config.ts` line 6) uses:
```typescript
loader: glob({ pattern: '**/*.{md,mdx}', base: './src/content/blog/great-loop' })
```
All MDX files placed in `src/content/blog/great-loop/` are automatically collected. No manifest file needed.
