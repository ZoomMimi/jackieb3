    Write scripts/import-blogger.mjs as a Node.js ESM script following the shebang + import header pattern from blog-viewer.mjs (shebang line, JSDoc block documenting --refresh and --force flags, import from node:fs / node:path / node:url, __dirname via dirname(fileURLToPath(import.meta.url))). Use node: protocol prefixes for all built-in imports. No CommonJS require().

    Constants to declare: BLOG_URL = 'https://jackiebiiigoingloopy.blogspot.com', FEED_BASE = template using BLOG_URL + /feeds/posts/default?alt=json&max-results=25, DATA_DIR = join(__dirname, '..', '.planning', 'data'), CACHE_PATH = join(DATA_DIR, 'blog-posts-full.json'), MDX_DIR = join(__dirname, '..', 'src', 'content', 'blog', 'great-loop'), REDIRECTS_PATH = join(__dirname, '..', 'public', '_redirects'), REFRESH = process.argv.includes('--refresh'), FORCE = process.argv.includes('--force').

    fetchAllPosts(): Extend the blog-viewer.mjs pagination loop verbatim — same startIndex=1, while(true), fetch with start-index and orderby=published, break on entries.length === 0, break on entries.length less than 25 at end of page. Per entry extract: title (entry.title.$t, default '(untitled)'), published (entry.published.$t.slice(0, 10) giving 'YYYY-MM-DD'), postUrl (entry.link find rel=alternate href), html (entry.content.$t — the full HTML body, NOT in blog-image-urls.json cache), labels (entry.category array filtered where scheme === 'http://www.blogger.com/atom/ns#', then map to .term), posterId (entry.id.$t). Return array sorted by date ascending. Note: do not capture images or videos separately — they remain in the raw html body.

    Cache-or-fetch pattern: if !REFRESH and existsSync(CACHE_PATH), load cached.posts from CACHE_PATH. Otherwise fetchAllPosts() and write CACHE_PATH as JSON with structure { generated: new Date().toISOString(), posts: [...] }. This matches blog-viewer.mjs cache convention.

    sanitizeHtml(html): Apply exactly 6 regex transforms in this order to avoid double-processing. (1) Blogger video iframes: match iframe elements whose src attribute contains blogger.com/video.g?token= and replace the entire iframe element with the comment string <!-- video: https://www.blogger.com/video.g?token={captured-token} --> — do NOT match or modify iframe elements whose src contains youtube.com/embed (those are already valid HTML and should be preserved as-is per RESEARCH.md Open Question 3). (2) Strip script blocks: match opening script tag through closing /script tag (multiline, case-insensitive), replace with empty string. (3) Strip style blocks: match opening style tag through closing /style tag (multiline, case-insensitive), replace with empty string. (4a) Self-close br void elements: match br tag with or without attributes not already self-closed, add space-slash before closing angle bracket. (4b) Self-close hr void elements: same pattern. (4c) Self-close img void elements: match img tag with all its attributes (including border="0", data-* attrs from Blogger WYSIWYG) not already self-closed, add space-slash — the regex must handle any attribute combination since Blogger generates img tags with many attributes. (5) Escape bare curly braces in text nodes: replace { not inside an HTML tag with &#123; and } not inside an HTML tag with &#125; — use the lookbehind approximation (?<!<[^>]*) as specified in Pattern 4; log a warning if any post triggers the 'Unexpected token' build failure so it can be fixed. Return trimmed result.

    extractSlug(postUrl): Parse postUrl with new URL(postUrl).pathname, split on '/', take last segment, strip .html suffix. Example: 'https://jackiebiiigoingloopy.blogspot.com/2022/04/getting-ready-to-go.html' -> 'getting-ready-to-go'.

    buildFrontmatter(entry): Return an object with: title = entry.title (string), date = entry.published (YYYY-MM-DD string), voyage = 'great-loop', location = entry.labels[0] if labels has entries otherwise 'Great Loop' (log to console any post using the 'Great Loop' fallback so Phase 3 can fill them in — expected for posts with no Blogger labels), excerpt = strip HTML from entry.html via /<[^>]+>/g replace, normalize whitespace, decode &amp; &lt; &gt; &nbsp; &quot; entities, trim, slice to 200 chars.

    buildMdx(frontmatter, sanitizedHtml): Generate the MDX string with YAML frontmatter block. Rules for YAML: double-quote ALL string values (title, voyage, location, excerpt); escape any " inside string values as \"; date field is bare YYYY-MM-DD without quotes (Zod coerces it via z.coerce.date()); migrated: true (bare boolean, no quotes); omit coverPhoto, lat, lon, anchorage, marina entirely. Body after the closing --- is the sanitizedHtml directly — no Markdown wrappers, no code fences, no added HTML tags. The rehype-raw plugin handles block-level HTML.

    buildRedirectLine(dateStr, slug): year = dateStr.slice(0,4), month = dateStr.slice(5,7). Return the string /year/month/slug.html /blog/dateStr-slug/ 301 (space-separated, no domain in either path, trailing slash on target path).

    Main loop: for each post in posts (sorted by date), compute slug = extractSlug(post.postUrl), filename = post.date + '-' + slug + '.mdx', outPath = join(MDX_DIR, filename). If !FORCE and existsSync(outPath), increment skipped counter and continue. Otherwise sanitize = sanitizeHtml(post.html), fm = buildFrontmatter(post), write buildMdx(fm, sanitize) to outPath, append buildRedirectLine(post.date, slug) to a redirectLines array, increment written counter.

    Write public/_redirects: write the comment header '# Blogger -> Astro 301 redirects (auto-generated by scripts/import-blogger.mjs)' then join redirectLines with newlines. Use writeFileSync to overwrite (not append) so re-runs produce clean output.

    Coverage validation (D-01 — XML substitute): load .planning/data/blog-image-urls.json, get its posts array. For each entry in blog-image-urls.json, extract its slug via the same extractSlug() function and check that join(MDX_DIR, entry.date + '-' + slug + '.mdx') exists. Log any missing slugs as coverage mismatches. Expected: 0 mismatches, 72 matches. Note: no Blogger XML export is on disk; if one is ever needed, download from Blogger admin → Settings → Manage blog → Back up content (per D-01).

    Console summary following blog-viewer.mjs style: posts loaded from API/cache, MDX files written, MDX files skipped, redirect lines written, label-fallback posts count, coverage mismatches count.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Blogger Atom API → import script | External HTTP response; entry.content.$t is untrusted HTML from own Blogger account |
| Blogger HTML → MDX file body | HTML sanitized before write; must not contain script/style/broken JSX |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | sanitizeHtml() — `<script>` blocks | mitigate | Step 2: strip all `<script>...</script>` blocks from Blogger HTML before writing MDX; prevents injected JS from executing in Astro build or browser |
| T-02-02 | Tampering | sanitizeHtml() — `<style>` blocks | mitigate | Step 3: strip all `<style>...</style>` blocks from Blogger HTML; prevents CSS injection into page layout |
| T-02-03 | Tampering | sanitizeHtml() — bare curly braces | mitigate | Step 5: escape `{` → `&#123;` and `}` → `&#125;` in text nodes; prevents MDX v3 treating Blogger prose as JSX expressions |
| T-02-04 | Tampering | sanitizeHtml() — Blogger video iframes | mitigate | Step 1: replace `blogger.com/video.g?token=` iframes with `<!-- video: ... -->` comment stubs (D-07); YouTube iframes preserved as valid trusted embeds |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan; implementation uses Node.js built-in modules only; no slopcheck required |
</threat_model>
