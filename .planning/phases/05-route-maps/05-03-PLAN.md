<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MDX frontmatter (title, excerpt, coverPhoto) -> build-time JSON island -> client popup HTML | Author-controlled strings are serialized into HTML and injected into Leaflet popup DOM |
| route-track.json -> build-time JSON island -> client | Build-produced GeoJSON injected into page HTML |
| client -> Stadia Maps tile CDN | Browser requests tiles from a third-party CDN |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Injection / XSS | popup HTML built by string interpolation of title/excerpt/coverPhoto | mitigate | Popup strings originate ONLY from build-time getCollection frontmatter (site owner authored); NO runtime/user input (URL params, location.*) is ever concatenated into popup HTML; coverPhoto used only as an `<img src>` attribute value |
| T-05-06 | Injection / XSS | `<script type="application/json">` data island breakout via `</script>` in a title/excerpt | mitigate | `serialize()` escapes `<` to `<` before `set:html`, preventing early script-tag termination; JSON.parse restores the value client-side |
| T-05-04 | Spoofing | Stadia Maps tile URL | accept | Hardcoded tile URL, no user input, no secrets transmitted; attribution rendered per license |
| T-05-SC | Tampering | npm installs | n/a | No new packages installed in this plan (leaflet already audited in RESEARCH.md) |
</threat_model>
