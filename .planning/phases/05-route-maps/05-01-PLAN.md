<objective>
Build the build-time GPX pipeline that produces the route polyline data consumed by the full-route map.

Purpose: MAP-02 requires the Great Loop GPX track drawn as a polyline. This plan writes the `scripts/simplify-gpx.mjs` converter (Nebo GPX -> simplified GeoJSON), commits a valid stub `src/data/route-track.json` so the build never fails before real GPX exists, and wires an `astro:build:start` hook so the polyline regenerates automatically on every build once GPX files are present.

Output: `scripts/simplify-gpx.mjs`, committed stub `src/data/route-track.json`, and an additive Vite/Astro integration hook in `astro.config.mjs`. A new dev dependency `@xmldom/xmldom` (DOM parser required by `@tmcw/togeojson` under Node — no global `DOMParser` exists in Node 24).

Note: The actual Nebo GPX export (D-10) is a human prerequisite handled by the finalization plan (05-04). This plan works with or without GPX files present.
</objective>

Map coordinates `[lon, lat]` to `{ x: lon, y: lat }`, call `simplify(pts, TOLERANCE, true)`, map back to `[x, y]` pairs. Build the output GeoJSON Feature (LineString) with the simplified coordinates. Ensure `src/data/` exists, then `writeFileSync(OUTPUT, JSON.stringify(output))` unless `--dry-run`. Log aligned lines: raw point count, simplified point count, and output KB. Compute `Buffer.byteLength(JSON.stringify(output), 'utf8')`; if it exceeds `SIZE_LIMIT`, `console.error` a message telling the operator to raise `TOLERANCE`, and `process.exit(1)`.

    Create the committed stub `src/data/route-track.json` now (identical shape to `writeStub()` output) so the build never fails on the VoyageMap import before real GPX exists.
