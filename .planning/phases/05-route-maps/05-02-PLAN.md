<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MDX frontmatter -> data-* attributes -> client Leaflet | Author-controlled lat/lon/location cross into a browser script |
| client -> Stadia Maps tile CDN | Browser requests map tiles from a third-party CDN |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | Tampering / Injection | lat/lon read from `data-*` attributes into Leaflet | mitigate | Values coerced with `Number()` and guarded by `Number.isFinite` before use; only numeric coordinates reach Leaflet APIs, never innerHTML |
| T-05-04 | Spoofing | Stadia Maps tile URL | accept | Tile URL is a hardcoded constant, no user input; no secrets sent (only z/x/y). Attribution rendered per license |
| T-05-08 | Information disclosure | `location` string in aria-label | accept | location is author-authored frontmatter; used as an accessibility label attribute value, not executable HTML |
| T-05-SC | Tampering | npm installs | n/a | No new packages installed in this plan (leaflet already audited in RESEARCH.md) |
</threat_model>
