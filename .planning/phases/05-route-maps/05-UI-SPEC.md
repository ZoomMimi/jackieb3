## Spacing Scale

All values are multiples of 4. Confirmed from Tailwind utility classes in existing Astro components.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, stat row gap (`gap-0.1rem`) |
| sm | 8px | Popup internal spacing, stat label-value gap |
| md | 16px | Default element spacing (`mb-4`) |
| lg | 24px | Section padding (`px-6`), nav gap |
| xl | 32px | Post mini map bottom margin (`mb-8`) — before body content |
| 2xl | 40px | VoyageMap bottom margin (`mb-10`) — separates map from post list |
| 3xl | 48px | Page vertical padding (`py-12`) |

Exceptions:
- **VoyageMap container height:** 400px exactly (D-02, locked). Not a spacing token — explicit height for map rendering.
- **PostMiniMap container height:** 192px (`h-48`). Not a spacing token — explicit height required for Leaflet initialization.
- **Popup max-width:** 280px. Popup min-width: 200px. Leaflet constraint, not page spacing.
- **Mobile touch targets:** 44px minimum height (`min-h-[44px]`) — existing pattern from Header.astro.

**Popup-specific overrides** (inline styles in Leaflet HTML string — outside CSS cascade):

**Accent (`#b91c1c`) reserved for — Phase 5 additions:**
- Stop marker SVG fill (12×12 circle on VoyageMap)
- Mini map pin SVG fill (14×14 circle on PostMiniMap)
- Popup "Read more" link text color
- (Pre-existing) Post title link hover color, active nav underline, blockquote border

**Polyline stroke:** `#1a2e4a` (navy) at `weight: 2, opacity: 0.7` — NOT the accent color. Matches the navy secondary role.

**className must be `''`** on all `L.divIcon()` calls — removes the default white-box `.leaflet-div-icon` background.

Source: codebase scan — `src/styles/global.css` (tokens), `src/components/Header.astro` (nav pattern), `src/layouts/BlogPost.astro` (link, blockquote colors). RESEARCH.md marker examples used `#8B6914` but that color does not exist in the design token system; `#b91c1c` is the correct accent.

## Component Inventory

Two new Astro components. No React, Svelte, or Vue — plain `.astro` files only.

### VoyageMap.astro

| Property | Value |
|----------|-------|
| File | `src/components/VoyageMap.astro` |
| Used on | `/voyages/great-loop/index.astro` only |
| Container class | `h-[400px] w-full mb-10` |
| Container style | `z-index: 0;` (inline — prevents Leaflet internal z-indices overriding nav) |
| Props | none (reads from `getCollection()` in frontmatter, imports route JSON) |
| Data delivery | Stop array: `<script type="application/json" id="voyage-stops">`, Route GeoJSON: `<script type="application/json" id="route-track">` |

| Initial view | `map.fitBounds(polyline.getBounds(), { padding: [30, 30] })` — never hardcode center |
| Fallback | If route GeoJSON is empty (no GPX processed), `fitBounds` over stop marker group instead |
| Data source | Filters `getCollection('blog')` to `voyage === 'great-loop' && !draft && lat !== undefined && lon !== undefined` |

### PostMiniMap.astro

| Property | Value |
|----------|-------|
| File | `src/components/PostMiniMap.astro` |
| Used on | Individual post pages only (`src/pages/blog/[...id].astro`) — never on list views |
| Placement | Above `<Content />` in BlogPost layout (D-06) |
| Container class | `h-48 w-full rounded mb-8` |
| Container style | `z-index: 0;` |
| Props | `lat: number, lon: number, location: string` |
| Data delivery | `data-lat` and `data-lon` attributes on container `<div>` |
| Map behavior | `dragging: false, zoomControl: false, attributionControl: true, scrollWheelZoom: false` |
| Initial view | `map.setView([lat, lon], 10)` |
| Conditional render | Rendered by `[...id].astro` only when `post.data.lat !== undefined && post.data.lon !== undefined` |
| Omission behavior | Posts without `lat`/`lon` show no mini map, no placeholder (D-08) |

### Route Data File

## Interaction Contracts

### VoyageMap — Full Route Map

| Interaction | Behavior |
|-------------|----------|
| Page load | Leaflet initializes, tiles load, polyline draws, `fitBounds` zooms to route automatically |
| Pan (drag) | Standard Leaflet pan — enabled |
| Zoom (scroll / buttons) | Standard Leaflet zoom controls — enabled (bottom-left corner, default Leaflet position) |
| Marker hover | No hover state (touch-friendly — no hover-only affordances) |
| Marker click / tap | Popup opens — `maxWidth: 280, autoPanPadding: L.point(20, 20)` |
| Popup content | Cover photo (80px tall, object-fit cover, lazy) + title (13px bold) + excerpt (11px muted) + "Read more →" link |
| Popup close | Click/tap outside popup or Leaflet close button |
| No GPX data | Map renders stop markers only, no polyline — graceful (not an error state) |
| No coverPhoto | Photo row is omitted from popup — no broken image, no empty box |
| Mobile pinch zoom | Enabled (Leaflet default) |
| Mobile popup tap | `autoPanPadding` auto-scrolls map so popup fits in viewport |

### PostMiniMap — Per-Post Mini Map

| Interaction | Behavior |
|-------------|----------|
| Page load | Map initializes with single pin at `[lat, lon]` zoom 10 |
| Pan | Disabled (`dragging: false`) — prevents scroll hijacking on mobile |
| Zoom | Disabled (`zoomControl: false, scrollWheelZoom: false`) — static location indicator |
| Attribution | Visible (required by Stadia Maps / OSM license) |
| Click/tap | No interaction — single static pin, no popup |
| Mobile scroll | Safe — `dragging: false` means touch scroll through the map works normally |

### Stacked Map Prevention

Only one Leaflet map instance per page. `VoyageMap` renders only on `/voyages/great-loop/index.astro`. `PostMiniMap` renders only on individual post pages with `lat`/`lon`. No page renders both. No list page renders any map. (Source: D-03, D-05, D-08; RESEARCH anti-pattern: "Multiple Leaflet maps on a list/index page.")

## Layout Constraints

### Voyage Index Page (`/voyages/great-loop/`)

```
<Layout>
  <div class="max-w-[720px] mx-auto px-6 py-12">
    <h1> Great Loop </h1>
    <p>  2022–2024 · 5,424 nm  </p>
    ← INSERT VoyageMap HERE (placeholder comment) →
    [post list]
  </div>
</Layout>
```

- Map is full-width within the `max-w-[720px]` wrapper
- `mb-10` (40px) separates map from post list
- Map does not break out of the container — no full-bleed treatment

### Individual Post Page (`/blog/[...id]`)

```
<BlogPost>            ← max-w-[720px] mx-auto px-6 py-12
  [post nav top]
  <article>
    <h1> title </h1>
    <time + location>
    ← INSERT PostMiniMap HERE (above <Content />) →
    <div class="post-body">
      <Content />
    </div>
  </article>
</BlogPost>
```

- Mini map `mb-8` (32px) separates it from the first paragraph of body content
- Mini map is full-width within the article, same width as prose

### Z-Index Stack

| Layer | z-index | Element |
|-------|---------|---------|
| Site nav (mobile dropdown) | 50 | `nav[aria-label="Main"]` in mobile details |
| Leaflet container | 0 (stacking context) | Map div |
| Leaflet popups | 600 (within map) | Relative to map stacking context |
| Leaflet markers | 400 (within map) | Relative to map stacking context |

The `z-index: 0` inline style on map container divs creates a stacking context that keeps all Leaflet internals (markers z-400, popups z-600) below the site nav z-50. This prevents map popups from appearing over the header. Source: CONTEXT.md code_context, RESEARCH.md pattern examples.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (popup) | "Read more →" — exact string, `→` character (not `&rarr;` in rendered text) |
| Voyage index empty state (no posts) | Heading: "No stops yet." Body: "Posts will appear here as they are published." (existing page has different copy — replace with this on Phase 5 refactor since the list is now below the map) |
| Map no-polyline state | Silent — no user-visible message. The map loads with stop markers. The absence of a polyline is not an error. |
| Map initialization failure | No UI error message displayed. Browser console error only. Map container remains as white/offwhite box. |
| Popup missing cover photo | No message — photo row is simply omitted. Title + excerpt + link render without it. |
| Mini map (no interaction) | No label or caption — the post's `location` field in the `<time>` meta row serves as the text description of the location. |
| Destructive actions | None in Phase 5. |

## Accessibility

| Requirement | Specification |
|-------------|--------------|
| Map container aria-label | `VoyageMap`: `aria-label="Interactive route map of the Great Loop voyage"`. `PostMiniMap`: `aria-label="Map showing the stop at {location}"` (location injected from prop). |
| Attribution link | Stadia Maps + OSM attribution links must be in the DOM and keyboard-accessible. Leaflet renders these automatically with `attributionControl: true`. |
| Popup keyboard access | Leaflet opens popups on `Enter`/`Space` on focused markers. Default Leaflet keyboard navigation applies. |
| Mini map keyboard | Container receives Leaflet's default focus ring. No custom keyboard interaction needed (static view). |
| Touch target (popup close) | Leaflet close button is 30×30px — meets 44px minimum only when paired with the surrounding popup area. Acceptable since tap-outside-to-close is the primary mobile gesture. |
| Color contrast | Accent `#b91c1c` on white `#ffffff` — ratio 5.08:1 (passes WCAG AA for normal text). Navy `#1a2e4a` on offwhite `#faf9f6` — ratio 13.4:1 (passes WCAG AAA). |
